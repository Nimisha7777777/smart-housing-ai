# Database Design - Smart Housing AI

## Overview

The Smart Housing AI platform follows a database-per-service architecture.

Each microservice owns its own database and is responsible for all CRUD operations on that data.

Other services must communicate through REST APIs instead of directly accessing another service's database.

# Design Principles

- Each service owns its own database.
- No service directly accesses another service's database.
- Communication between services is done using REST APIs.
- MySQL is used for structured relational data.
- MongoDB is used for flexible visitor records.
- Financial transactions are isolated inside the Payment Service.
-Each business entity belongs to exactly one society.
-Data isolation is achieved using the societyId present in every business entity.
-Services never directly access another service's database.
-All inter-service communication happens via REST APIs.
